# HTTP/2

- Binary
- HPACK
- TCP
- TLS
- Server PUSH

バイナリベースのプロトコルに変更された。 フレームという単位でデータを送受信する。

1つのTCPコネクションの中にストリームという仮想TCPソケットを作成して並列でデータの送受信がされる。

HPACKでヘッダ圧縮する。

# HTTP/3

- QPACK
- UDP(QUIC)
- TLS/1.3
- Server PUSH

QUICというUDPを用いたプロトコル上で動作する。 QUICはトランスポート創の話。

HPACKの代わりにQPACKでヘッダ圧縮する。
